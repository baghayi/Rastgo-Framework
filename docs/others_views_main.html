بخش های دیگر

این پوشه هم برای قرار دادن قالب ها مون هست و فایل هایی که باید توسط کلاس قالبمون رندر بشه

برای اینکه بشه برای سیستم فالب های متنوع ای درست کرد و هر موقع خواسته شد بدون مشکل بشه تغییرشون داد / بجای اینکه همه فایل ها تو همین پوشه View قرار بگیرن طوری درست کردم که هر قالب یه پوشه و نامی داشته باشه و همه فایل هایی که قراره رندر بشن و به همراه کدهای css و جاوااسکریپت و عکس ها تو همون پوشه قرار بگیره

که اسم قالب پیش فرض این فریمورک defaultTemplate هستش

که این پوشه defaultTemplate سه تا پوشه داره به نام های images که عکس های قالب توش قرار بگیرن یا هر عکس دیگه ای و بعد پوشه javascript برای قرار دادن فایل های جاوااسکریپتمون و آخری هم stylesheets برای قرار دادن فایل های css مون

البته خودتون هم می تونید پوشه هایی ایجاد کنید و مشکلی نخواهد بود و هر نامی هم باشه مشکلی ایجاد نمی شه و نیازی هم نخواهد بود در فریمورک تغییری بدید

فایل آخری موجود در defaultTemplate هم index.php هستش و یه ساختار html ای داره و چند تا متغیر توشون تعریف شده برای (تست) و رندر شدن (که در مورد رندر و نحوه قرار دادن متغیر ها و فلان و بهمان قبلا بحث ششده)

توجه کنید که می تونید داخل فایل های قالب مثل index.php در پوشه defaultTemplate دقیقا مثل php عکس کرده و از foreach و ... استفاده کنید و مشکلی ایجاد نخواهد شد (که بستگی به نحوه رندر کردن کلاس تون داره که برای این فریمورک نحوه رندر کردنش توضیح داده شده و اگه با کلاس خودتون جایگزین کنید نسبت به کار کلاس خودتون این هم باید تغییر پیدا کنه طبیعتا!)

نکات

یه نکته ای که موند اینه که برای لود کردن فایل های css و js و ... ممکنه نیاز داشته باشید که آدرس کاملش رو وارد کنید که برای این کار می تونید از ثابت های تعریف شده برای این کار در فایل bootStrap.php استفاده کنید (یبار ثابت ها رو بخونید و ببینید کارشون چیه / که در توضیحات همین فایل هم در مورد ثابت ها بحث شده) که می تونید از اونا استفاده کنید.

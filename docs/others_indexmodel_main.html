بخش های دیگر

کلاس indexModel هم که نمونه ای از model های ماست (و به اسم index که بطور پیشفرض باید به این اسم باشه / مگر اینکه اسم کنترلر رو از index به اسم دیگه ای تعییر بدید که در موردش قبلا بحث شده/ در توضیحات فایل application\controllers\indexController.php )

که اسم model هم در حقیقت index هست که بنا به قوانین این فریمورم باید کلمه Model رو به انتهای model اضافه کنید (هم برای فایل model و هم برای اسم کلاسش)

وظیفه model ها هم اتصال به دیتابیس و انجام عملیات مورد نظر ماست

و همینطور این کلاس از کلاس baseModel ارث بری می کنه (برای اینکه بتونیم از دیتابیس استفاده کنیم) در صورتی که نیاز به دیتابیس ندارم می تونیم ارث بری نکنیم ازش (به این توجه کنید که کار model ها کار با دیتابیس هست پس در نتیجه ارث بری نکردن ازش بی معنی خواهد بود)


متدها

متد پیش فرض کلاس index هست که چون این متد abstract در کلاس پدر هست (baseModel) پس باید تعریف بشه

و همینطور متنی داخل این متد فقط برای تست آورده شده است که کارش نشون داده بشه و در تست سربلند بیاد بیرون :))

یه متد دیگه ای به این کلاس اضافه شده به اسم cache که نحوه استفاده از کلاس کش توش بکار رفته و به عنوان مثال افزوده شده و کار خاصی نمی کنه

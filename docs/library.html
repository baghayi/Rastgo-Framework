هسته فریمورک

بخش های دیگر

تمام کلاس های فریمورک در پوشه ای بنام library در شاخه اصلی قرار دارن و هر کلاس دارای یک پوشه مجزا به خود با نام خود کلاس خواهد بود، که اطلاعات مربوط به کلاس، از جمله خود کلاس، توضیحات، وابستگی ها، نمونه کدهای ایجاد شده برای کلاس می تونه داخل پوشه مربوط به هر کلاس قرار بگیره.

برای استفاده از کلاس ها و در حقیقت صدا زدنشون باید از namespace ها شون استفاده بشه، پس در نتیجه تعیین namespace برای هر کلاس ضروری می باشد و namespace ها مثل شاختار پوشه ها تعیین می شن.

مثلا شاخه اصلی فریمورک بنام root هست و بعد که وارد شاخه library می شیم و سپس وارد پوشه کلاس و در نهایت نام خود فایل (فایلی ای که کدها درش قرار دارن، بدون پسوند فایل) رو به عنوان namespace تعیین می کنید.

نکته: پسوند فایل ها هم باید .php باشن که نبایستی در namespace قرار بگیره.

که به عنوان مثال می شه:

namespace root\library\YOUR_CLASS_FOLDER_NAME\YOUR_CLASS_FILE_NAME_WITHOUT_EXTENSION;


نحوه نام گزاری پوشه های هر کلاس

کلاس ها رو همینطوری نباید قرار بدید و یه پوشه ای ایجاد می کنید دقیقا به اسم نام کلاس (داخل فایل class CLASSNAME{}) و به کوچکی و بزرگی حروف هم دقت کنید که برابر باشن.

داخل پوشه هم نام فایل کلاس رو هر چی قرار بدید باید همون رو در انتهای namespace قرار بدید، مثلا نام فایل رو بزارید index.php کلمه index رو هم باید به namespace تو داخل فایل کلاس اضافه کنید.

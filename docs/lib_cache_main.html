index.php (کلاس اصلی)

این کلاس همانطوری که از اسمش پیداست برای کش کردن استفاده می شه.

پروپرتی ها

- $fileExtension برای ذخیره پسوند فایل کش

- $hashFileName مقدار true یا false می گیره و در صورت true بودن فایل کش رو هش خواهد شد

- $cacheFolderPath که آدرس کامل پوشه کش رو توش ذخیره می کنه

- $cacheFolderName که اسم پوشه کش رو توش ذخیره می کنه

- $bufferContent که اگه از توابع فافر کلاس استفاده کنید، محتوا تو این پروپرتی دخیره خواهد شد

- $hasBufferStarted هم که در صورتی که فافر شروع شده باشه مقدار پیش فرضش که false هست به true تغیر می کنه و بعد از اتمام برمی گیرده به سرجاش


متدهای کلاس

- اولین متد کلاس، مند سازنده هست و همون ابتدا آدرس کامل پوشه کش رو ایجاد و در پروپرتی مناسبش ذخیره می کنه

متد بعدی و از مهمترین ها که بیشتر باهاش سروکله خواهیم زد cacheContent و پابلیک هست هستش و ۴ تا پارامتر قبول می کنه

پارامتر اول نام کش - پارامتر دوم نام گروه که به نام کلاس اضافه می شه و فایله ایجاد می شه و با خط تیره از هم جدا خواهند بود - پارامتر سوم محتوایی که می خواییم کش بشن - پارامتر چهارم هم زمان کش شدن، یعنی تا چند وقت از الان کش مون اعتبار داشته باشه و برحسب ثانیه است

این متد پارامتر ها رو می گیره و سریالیز کرده و و در فایلی در پوشه کش ذخیره می کنه

- متد بعدی createCacheFolder و پرایوت هستش و وظیفه ایجاد پوشه کش رو در پوشه __rfolder داره

- متد بعدی getCache و از مهمترین متدها مون و پابلیک هم هستش و ۲ تا پارامتر می گیره

پارامتر اولی نام فایل کش و دومی هم نام گروه فایل کش هست و همان مقادیری هستش که در متد cacheContent داده شده بود که فایل کش ایجاد شده بود و برای گرفتن و استفاده از فایل کش هم باید همان مقادیر وارد شد.

- متد مهم دیگه مون startBuffer و پابلیک هستش.

اگه بجای اینکه محتوا رو بدید به متد مربوطه که قبلا در مورد متد بحث کردیم و می خواهید محتوا رو بافر کنید و بافر رو ذخیره کنید از همین متد می تونید استفاده کنید

هیچ پارامتری قبول نمی کنه و از جایی که می خواهید بافر شروع بشه صداش می کنید

- متد بعدی مون cacheBuffer و مکمل متد قبلی مون هستش و پابلیک هم هست

برای ذخیره بافرمون که از بعد از متد قبلی باید صدا زده بشه و ۳ تا پارامتر قبول می کنه

پارامتر اولی نام فایل کش - پارامتر دومی نام گروه فایل کش - پارامتر سومی هم زمان ای که فایل کش اعتبار خواهد داشت و برحسب ثانیه هستش

این متد علاوه بر ذخیره محتوای بافر مون، همون محتوا رو هم بازکشت می ده و می شه ازش استفاده کرد

- متدی بعدی deleteCache و پابلیک هم هست و برای حذف دستی فایل کش ای که می خواهید حذف کنید مورد استفاده قرار می گیره و ۲ تا پارامتر می گیره

پارامتر اولی نام فایل کش - و پارامتر دومی هم نام گروه فایل کش هستش

- متد بعدی مون deleteAGroup نام داره و پابلیک هستش و تنها یک پارامتر قبول می کنه

وظیفه این متد اینه که تمام فایل هایی که دارای اسم گروه یکسانی هستند رو یکجا حذف می کنه و مهم نیست اسم فایل ها چی باشه و فقط حسب اسم گروه حذفشون می کنه

تنها پارامتری که می گیره نام گروه فایل کش هستش

- متد بعدی setFileExtension و پابلیک هستش و باهاش می شه پسوند فایل کش رو تعیین کرد که بطور پیش فرض .txt هستش و یک پارامتر قبول می کنه و اونم پسوند فایل رو باید باهاش تعیین کنید و نقطه هم باید حتما بزارید

مثلا:

.log
.txt
...

- متد بعدی getFileExtension نام داره و پابلیک هستش و برای کرفتن پسوند فایل تعیین شده هستش

- متدی بعدی مون setCacheFolderName نام داره و پابلیک هستش و برای تعیین کردن نام پوشه کش استفاده می شه و بطور پیش فرض cache تعیین شده

وتنها پارامتری که می گیره و باهاش نام پوشه رو تعیین می کنید.

- متد بعدی مون getCacheFolderName نام داره و پابلیک هستش و برای گرفتن نام پوشه کش ای که تعیین شده استفاده می شه

- متد بعدی getCacheFolderPath نام داره و پابلیک هستش و برای کرفتن آدرس کامل پوشه کش مورد استفاده قرار می گیره

آدرس ای که در متده سازنده کلاس تعیین شده بود و در پروپرتی مناسبش ذخیره شده بپد

- متد بعدی setHashFileName نام داره و پابلیک هستش

بطور پیش فرض مقدار پرودرتی که مربوط به این مورد می شه true هستش و باعث می شه هر اسم فایل و اسم گروه فایل کش ای که وارد می شه بصورت هش شده باشه و با md5 هش می شه و در صورتی که به این متد مقدار false رو بدید بدون هش شده فایل کش ذخیره خواهد شد

تنها پارامتری که قبول میکنه بولین باید داده بشه و true یا false

با استفاده از متد getHashFileName هم که پابلیک هستش می تونید بفهمید قراره اسم فایل هاتون هش بشه (که true هست) که قرار نیست هش بشه (که false هست)

متد بعدی مون encryptName نام داره و پیرایوت هستش و نام فایل کش و نام گروه فایل کش رو می گیره و برحصب مقدار تعیین شده که اسم ها رو هش کنه یا نه عمل می کنه و مقدار رو می ده بیرون

نام فایل کش با یک خط تیره با نام گروه فایل کش جدا می شه (underline) ـ و سپس پسوند فایل که با نقطه شروه می شه هم به این مورد اضافه شده و بازگشت داده می شه.

بخش های دیگر

این فایل هم دیگه کلاس نیست! بلکه ثابت ها و فایل هایی که حتما باید ابتدای کار لود بشن رو تعریف و لود کرده و همینطور مقدار دهی شدن.


ثابت ها

ثابت اولی DS هست که مخفف شده ثابت DIRECTORY_SEPARATOR هستش که فایل ها و فولدورها مون رو از هم با این ثابت جدا کنیم که تو ویندوز و لینوکس این بک اسلش و فوروارد اسکش هر کدومشون بسته به سیستم عامل وارد بشن!

ثابت بعدی FILE_PATH نام داره که آدرس ریشه فریمورک رو تعیین می کنه که برای فایل ها و پوشه ها مون بتونیم از آدرس کامل استفاده کنیم (به جای ۲ نقطه بیا عقب / یه تا برو جلو و ... )

ثابت بعدی مون URL نام داره که آدرس کامل سایت (URL - مثلا http://test.com/framework) رو درست می کنه که بشه ازش در جاهایی خاص خودش استفاده بشه.

گاها نیازه مثلا url ای بدیم برای لود چیزی / مثل عکس ها و ... .

ثابت بعدی مون TEMPLATE_URL هست که آدرس URL برای تا پوشه قالب رو برامون آماده می کنه که ازش می تونیم تو قالب برای لود فایل های css یا JS یا ... استفاده کنیم


فایل های لود شده

تو این فایل دو تا فایل های config هم لود شدن:

config/config.php
config/database.php

تو config.php هم ثابت هایی تعریف شده که برای هر پروژه متغییر خواهد بود که در موردش بحث می کنیم

و همینطور database.php هم کلاسی برای تعیین کردن مقادیر دیتابیس هستش که در مورد این هم بحث می کنیم


و در نهایت کلاس Loader رو لود کردیم که بتونه کلاس های دیگه رو لود کنه و خط بعدش توسط متد استاتیک setAutoLoader() تابع autoload ای که شبیه سازی کرده بودیم رو به PHP معرفی می کنیم که برای لود کلاس ها دنبال ابن تابع بیاد بجای __autoload(){}
